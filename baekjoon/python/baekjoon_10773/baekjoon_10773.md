# [백준/Python] 10773번: 제로

### 문제 링크
[백준 10773번: 제로](https://www.acmicpc.net/problem/10773)

### 문제 설명
재현이가 부르는 숫자를 장부에 적거나 지우는 문제입니다.
0보다 큰 정수가 주어지면 장부에 해당 숫자를 적고, 0이 주어지면 가장 최근에 쓴 수를 지웁니다. 모든 숫자를 입력받은 후, 장부에 남아있는 수들의 합을 구하여 출력해야 합니다.

### 제출 코드

```python
import sys

def calculate_ledger():
    k = int(sys.stdin.readline())
    stack = []
    
    for _ in range(k):
        num = int(sys.stdin.readline())
        
        if num == 0:
            stack.pop()
        else:
            stack.append(num)
            
    print(sum(stack))

if __name__ == '__main__':
    calculate_ledger()
```

### 코드 분석

이 문제는 후입선출(LIFO, Last-In-First-Out) 구조를 가지는 스택(Stack) 자료구조를 활용하는 전형적인 문제입니다. 

**1. 입력 처리 최적화**
```python
import sys
k = int(sys.stdin.readline())
```
입력 데이터의 개수 K가 최대 100,000까지 주어질 수 있습니다. 반복문 내부에서 `input()`을 지속적으로 호출할 경우 시간 초과가 발생할 위험이 크므로, `sys.stdin.readline()`을 사용하여 입출력 속도를 최적화했습니다.

**2. 스택을 활용한 데이터 삽입 및 삭제**
```python
if num == 0:
    stack.pop()
else:
    stack.append(num)
```
파이썬의 리스트(List)는 스택의 역할을 완벽하게 수행할 수 있습니다. 
입력받은 숫자가 0이 아닐 때는 `append()` 메서드를 사용하여 리스트의 마지막에 값을 추가합니다. 반대로 0이 입력되었을 때는 `pop()` 메서드를 사용하여 리스트의 가장 마지막 요소(가장 최근에 쓴 수)를 제거합니다. 문제의 조건에서 "정수가 0일 경우에 지울 수 있는 수가 있음을 보장한다"고 명시되어 있으므로, 스택이 비어있는지 확인하는 예외 처리 구문은 생략하여 코드를 간결하게 유지했습니다.

**3. 결과 출력**
```python
print(sum(stack))
```
파이썬의 내장 함수인 `sum()`을 활용하면 반복문을 돌며 값을 더할 필요 없이, 리스트 내부에 남아있는 모든 요소의 합을 O(N)의 시간 복잡도로 빠르게 계산할 수 있습니다.

### 파이썬 최적화 및 팁

스택 구조에서 데이터를 추가하고 삭제하는 과정은 리스트의 끝부분에서 발생합니다. 파이썬 리스트의 `append()`와 `pop()` 연산은 시간 복잡도가 O(1)이므로, K번의 반복문 동안 O(K)의 시간 복잡도를 가집니다. 마지막에 `sum()` 함수를 실행하는 시간 역시 O(K)이므로, 전체 프로그램의 시간 복잡도는 O(K)로 매우 효율적입니다.

만약 이와 비슷한 문제에서 가장 앞의 원소를 빼야 하는 상황(큐 구조)이 발생한다면 리스트의 `pop(0)`은 O(N)의 시간이 걸리므로, 이때는 `collections` 모듈의 `deque`를 사용해야 시간 초과를 피할 수 있습니다. 이번 문제는 스택의 기본에 충실한 문제이므로 리스트만으로 충분히 최적의 성능을 낼 수 있습니다.
