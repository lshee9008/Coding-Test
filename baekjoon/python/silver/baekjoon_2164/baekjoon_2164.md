# [백준/Python] 2164번: 카드2

### 문제 링크
[백준 2164번: 카드2](https://www.acmicpc.net/problem/2164)

### 문제 설명
1부터 N까지의 번호가 붙어있는 N장의 카드가 있습니다. 1번 카드가 제일 위에, N번 카드가 제일 아래에 있는 상태로 순서대로 놓여 있습니다.
카드가 단 한 장 남을 때까지 아래의 동작을 반복합니다.
1. 제일 위에 있는 카드를 바닥에 버린다.
2. 그 다음 제일 위에 있는 카드를 제일 아래에 있는 카드 밑으로 옮긴다.

최종적으로 마지막에 남게 되는 카드의 번호를 구하는 문제입니다.

### 제출 코드

```python
import sys
from collections import deque

def find_last_card():
    # 카드의 개수 N 입력
    n = int(sys.stdin.readline())
    
    # 1부터 N까지의 숫자를 덱(deque)에 초기화
    cards = deque(range(1, n + 1))
    
    # 카드가 1장 남을 때까지 반복
    while len(cards) > 1:
        # 1. 제일 위에 있는 카드를 버림
        cards.popleft()
        
        # 2. 그 다음 제일 위에 있는 카드를 뽑아서 제일 아래로 옮김
        cards.append(cards.popleft())
        
    # 마지막에 남은 카드 1장 출력
    print(cards[0])

if __name__ == '__main__':
    find_last_card()
```

### 코드 분석

이 문제는 선입선출(FIFO, First-In-First-Out) 구조인 **큐(Queue)** 를 활용해야 합니다. 파이썬에서는 큐 연산을 최적화하기 위해 `collections` 모듈의 `deque`(Double-Ended Queue)를 사용하는 것이 정석입니다.

**1. deque 자료구조의 활용**
```python
from collections import deque
cards = deque(range(1, n + 1))
```
리스트(List) 대신 `deque`를 사용한 이유는 **시간 복잡도** 때문입니다. 
파이썬의 리스트에서 맨 앞의 요소를 제거하는 `pop(0)` 연산은 남은 모든 요소를 한 칸씩 앞으로 당겨야 하므로 $O(N)$의 시간이 소요됩니다. N이 최대 500,000이므로 리스트를 사용하면 반복문 내부에서 계속 $O(N)$ 연산이 발생하여 시간 초과(TLE)가 발생합니다.
반면 `deque`는 양방향에서 데이터를 처리할 수 있도록 설계되어 있어, 맨 앞의 요소를 제거하는 `popleft()` 연산을 $O(1)$의 시간 복잡도로 빠르게 수행할 수 있습니다.

**2. 문제의 동작 로직 구현**
```python
while len(cards) > 1:
    cards.popleft()
    cards.append(cards.popleft())
```
카드가 한 장 남을 때까지 반복하는 `while` 루프 안에서 문제에 제시된 두 가지 동작을 직관적으로 구현했습니다.
첫 번째 줄의 `popleft()`로 맨 위의 카드를 버리고, 두 번째 줄에서는 다시 `popleft()`로 뽑아낸 카드를 `append()`를 통해 덱의 맨 뒤(아래)로 삽입합니다.

### 💡 파이썬 최적화 및 수학적 규칙 팁

`deque`를 사용하는 것이 출제자가 의도한 정석적인 풀이 방식입니다. 하지만 이 문제는 카드 번호의 변화 과정에 숨겨진 **수학적 규칙**을 찾으면 반복문 없이 $O(1)$의 시간 복잡도로 단번에 정답을 도출할 수도 있습니다.

N에 따른 마지막 카드의 결과를 나열해 보면 2의 제곱수를 기준으로 규칙이 초기화되는 것을 알 수 있습니다.
* N이 2의 제곱수(2, 4, 8, 16...)일 때는 항상 마지막 카드가 N이 됩니다.
* 그 외의 경우에는 N보다 작은 가장 큰 2의 제곱수를 뺀 값에 2를 곱한 수가 정답이 됩니다.

이를 파이썬 코드로 구현하면 아래와 같이 매우 짧게 작성할 수 있습니다.

```python
import sys
import math

n = int(sys.stdin.readline())

if n == 1:
    print(1)
else:
    # n보다 작은 가장 큰 2의 제곱수(제곱 지수)를 구함
    power = 2 ** int(math.log2(n - 1))
    print((n - power) * 2)
```

알고리즘 테스트에서는 직관적이고 안정적인 `deque`를 사용한 자료구조 풀이를 우선적으로 권장하며, 수학적 규칙을 찾는 풀이는 코딩 테스트나 면접에서 성능을 극한으로 끌어올려야 할 때 추가적인 어필 용도로 알아두시면 좋습니다.
